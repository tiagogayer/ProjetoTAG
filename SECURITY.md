# Security Policy

## Supported Versions

This project does not follow a versioned release schedule.
The latest commit on `main` is the only supported version.

When reporting, identify the exact state you are running —
include the commit SHA or version tag if available.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

Instead, use [GitHub's private vulnerability reporting](https://github.com/tiagogayer/ProjetoTAG/security/advisories/new)
for this repository.

When reporting, please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof of concept
- The affected component or configuration
- Any suggested mitigations, if known

You can expect an acknowledgment within 3 business days.
After acknowledgment, findings will be assessed and next steps communicated.
If confirmed, a GitHub Security Advisory will be published once the fix is in place.
